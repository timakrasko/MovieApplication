# Linting в проєкті

## Обраний лінтер та причини вибору

### **Detekt**
Для аналізу коду Kotlin у цьому проєкті використовується **Detekt**. Причини вибору:
- **Глибокий аналіз коду**: перевіряє стиль, безпеку та складність коду.
- **Гнучкість налаштувань**: дозволяє вимикати або модифікувати правила відповідно до вимог проєкту.

## Правила та їх пояснення

### **Основні правила Detekt**
В роботі було використано базові правила, довелося лише внести декілька змін, які не відповідали вимогам найменування Compose функцій

## Інструкція з запуску лінтера

```sh
./gradlew detekt
```
